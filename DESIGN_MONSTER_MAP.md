# 弹珠赌局 · 改版设计方案：地图即怪物

> 本文档为设计方案，不修改任何代码。基于现有 PinballScene.ts 机制整理。

---

## 一、核心理念

**整个弹珠桌就是一个活体怪物。** 玩家不再是"击败上方Boss"，而是用弹珠一点一点地摧毁怪物的各个器官——眼睛、脊椎、肋骨、关节、血肉壁——直到核心（心脏）暴露并被击碎。

每一次碰撞都是对怪物身体的破坏，每一个障碍物都有独立HP，被摧毁后从地图上消失，改变球的运动路径。地图随着战斗推进而"崩塌"，路径不断变化，越打越开放。

---

## 二、三种弹珠款式

玩家在发射台蓄力时，按 `1`/`2`/`3` 键切换弹珠款式。每种款式有独立数量，用完即不可选。

### 2.1 基础弹珠（现有款式，保留）

| 属性 | 值 | 说明 |
|------|-----|------|
| 颜色 | 白色 `0xeeeeee` | — |
| 半径 | 12 | 与现有一致 |
| 弹力 | 0.98 | 高弹性 |
| 耐久 | 28 | 标准耐久 |
| 对器官伤害 | 1 / 次碰撞 | — |
| 特殊机制 | 无 | 纯粹的物理碰撞 |
| 数量 | 5 颗 | — |
| 切换键 | `1` | — |

**定位**：全能型，弹力高、耐久适中，适合在地图中反复弹射累积伤害。没有特殊效果但最稳定。

### 2.2 烈焰弹珠（新增）

| 属性 | 值 | 说明 |
|------|-----|------|
| 颜色 | 橙红 `0xff6600`，带发光描边 | — |
| 半径 | 14 | 比基础大，视觉上更"重" |
| 弹力 | 0.95 | 弹力略低，能量衰减更快 |
| 耐久 | 20 | 耐久较低，"烧"得快 |
| 对器官伤害 | 2 / 次碰撞 | 双倍伤害 |
| 特殊机制 | **点燃** | 碰撞时点燃器官，持续灼烧3秒，每秒-1HP |
| 数量 | 3 颗 | — |
| 切换键 | `2` | — |

**点燃机制详细说明**：
- 烈焰弹珠碰撞器官时，除即时造成2点伤害外，给器官附加 `burnTimer = 3000ms` 状态
- 每帧 `burnTimer` 递减，每满1000ms对器官造成1点伤害
- 点燃期间器官视觉闪烁橙红色
- 点燃效果可叠加刷新（再次碰撞重置为3000ms），但伤害不叠加（每秒固定-1）
- 被点燃的器官如果正在运作（如眼睛发射子弹），运作不受影响——灼烧只扣HP

**定位**：高伤爆发型，适合快速摧毁高HP器官（肋骨8HP、血肉壁10HP）。代价是耐久低、弹力低，存活时间短，需要精准瞄准。

### 2.3 寒冰弹珠（新增）

| 属性 | 值 | 说明 |
|------|-----|------|
| 颜色 | 冰蓝 `0x66ccff`，带结晶描边 | — |
| 半径 | 10 | 比基础小，更灵活 |
| 弹力 | 1.00 | 极高弹性，几乎不衰减 |
| 耐久 | 35 | 耐久最高，"冰"得久 |
| 对器官伤害 | 1 / 次碰撞 | 标准伤害 |
| 特殊机制 | **冻结** | 碰撞时冻结器官2秒，冻结期间器官完全停止运作 |
| 数量 | 3 颗 | — |
| 切换键 | `3` | — |

**冻结机制详细说明**：
- 寒冰弹珠碰撞器官时，除即时造成1点伤害外，给器官附加 `freezeTimer = 2000ms` 状态
- 冻结期间器官**完全停止运作**：
  - 眼睛：停止发射子弹
  - 关节（旋转挡板）：停止旋转
  - 脊椎骨：停止冲击波发射
  - 肋骨/血肉壁：碰撞不再对球造成耐久伤害（但仍会物理弹射球）
- 冻结期间器官视觉变为冰蓝色
- 冻结效果不可叠加刷新（再次碰撞只重置为2000ms，不延长）
- 冻结不影响灼烧（两种状态可同时存在）

**定位**：控制生存型，伤害低但存活极久，适合冻结危险器官（尤其是发射子弹的眼睛）来争取安全输出时间。高弹力+高耐久=长时间在场，但清场速度慢。

### 2.4 三种弹珠对比

```
         伤害  耐久  弹力  特殊      定位
基础      1     28    0.98  无       全能稳定
烈焰      2     20    0.95  点燃DoT  高伤爆发
寒冰      1     35    1.00  冻结停止  控制生存
```

**策略循环**：
- 开局用寒冰冻结眼睛→争取安全时间用基础弹珠清理脊椎→用烈焰弹珠集中摧毁高HP肋骨/血肉壁→核心暴露后用烈焰弹珠爆发收尾

### 3.1 整体分区

├──────────────────────────────────────────┤
│  y=120~280 脊椎区                          │  ← 中路封锁

### 3.2 器官详细设计

#### 👁 眼睛（2个）— 对应现有 Boss 系统

| 属性 | 值 |
|------|-----|
| 位置 | 左眼(200,80) 右眼(600,80) |
| HP | 各5 |
| 半径 | 35（比原Boss略小） |
| 攻击 | 每2秒发射1颗子弹，子弹速度300，子弹伤害2耐久 |
| 被摧毁后 | 永久停止发射子弹，视觉消失，该侧上方开放 |
| 分数 | 300/个 |
| 映射现有代码 | `createBoss` → `createEyes`，`hitBoss` → `hitEye`，`fireBullet` |

**子弹调整**：子弹伤害从5（第三关）降为2，因为现在有两种弹珠+更多器官，整体难度已提升。子弹速度保持300。

#### 🦴 脊椎骨（3节）— 对应现有 Nail 系统

| 属性 | 值 |
|------|-----|
| 位置 | (400,150)(400,200)(400,250) 纵向排列 |
| HP | 各3 |
| 半径 | 10（比原nail略大） |
| 机制 | 物理碰撞弹射球，每次碰撞-1球耐久+1器官伤害 |
| 特殊攻击 | 每5秒发射一次"冲击波"——以脊椎骨为中心向四周推开球（速度200，持续0.5秒） |
| 被摧毁后 | 从地图消失，中路打开，停止冲击波 |
| 分数 | 50/节 |
| 映射现有代码 | `createNail` → `createSpine`，`hitNail` → `hitSpine` |

**冲击波机制**：脊椎骨存活时，每5秒发出一圈可见的环形波纹（视觉提示），将范围内的球向外推开。这增加了中路的风险——即使球没碰到脊椎骨，也可能被冲击波改变轨迹。

#### 🫁 肋骨（4根）— 对应现有 Bumper 系统

| 属性 | 值 |
|------|-----|
| 位置 | 左上(150,330) 左下(150,380) 右上(650,330) 右下(650,380) |
| HP | 各8 |
| 半径 | 28 |
| 机制 | 碰撞时强力弹射球（bounce 450），每次碰撞-1球耐久+1器官伤害 |
| 被摧毁后 | 从地图消失，侧路打开 |
| 分数 | 100/根 |
| 映射现有代码 | `createBumper` → `createRib`，`hitBumper` → `hitRib` |

**设计意图**：肋骨是高HP坦克型器官，基础弹珠需要8次碰撞才能摧毁，烈焰弹珠配合点燃只需约4次碰撞+3秒灼烧。肋骨弹力强，会把球弹向中央脊椎区，增加中路风险。

#### ⚙ 关节（2个）— 对应现有 Spinner 系统

| 属性 | 值 |
|------|-----|
| 位置 | 左(250,300) 右(550,300) |
| HP | 各6 |
| 机制 | 旋转挡板，手动碰撞检测，碰撞时-1球耐久+1器官伤害+切向弹力 |
| 被冻结时 | 停止旋转（碰撞仍有效但不旋转） |
| 被摧毁后 | 从地图消失，停止旋转 |
| 分数 | 150/个 |
| 映射现有代码 | `createSpinner` → `createJoint`，`updateSpinners` → `updateJoints` |

#### 🩸 血肉壁（2块）— 对应现有 Grotesque 系统

| 属性 | 值 |
|------|-----|
| 位置 | 左(60,450,w=100,h=20) 右(740,450,w=100,h=20) |
| HP | 各10 |
| 机制 | 碰撞时向上弹射球（bounce 420），每次碰撞-1球耐久+1器官伤害 |
| 被摧毁后 | 从地图消失，底部两侧开放（球更容易掉下去） |
| 分数 | 100/块 |
| 映射现有代码 | `createGrotesque` → `createFleshWall`，`hitGrotesque` → `hitFleshWall` |

**设计意图**：血肉壁是最后的防线，摧毁后底部两侧开放，球更容易从侧面掉落。玩家需要权衡：摧毁血肉壁能获得分数和进度，但也会让自己的防线更脆弱。

#### ❤️ 心脏（1个）— 最终目标

| 属性 | 值 |
|------|-----|
| 位置 | (400,530) |
| HP | 30 |
| 初始状态 | **不可攻击**——有保护层，球碰撞时只弹开不造成伤害 |
| 暴露条件 | 其他13个器官中摧毁≥10个（约75%）时，保护层碎裂，心脏暴露 |
| 暴露后 | 心脏开始跳动（视觉脉动），可被球攻击，每次碰撞-1HP |
| 被摧毁后 | `clearLevel()` → 通关 |
| 分数 | 1000 |
| 映射现有代码 | 新增 `createHeart`，`hitHeart`，在 `clearLevel` 中触发胜利 |

**心脏暴露机制详细说明**：
- 初始时心脏有一个半透明保护罩（视觉），球碰到会弹开但不造成伤害
- 当 `destroyedOrganCount >= 10`（共13个器官）时：
  - 保护罩碎裂动画（粒子效果或简单闪烁）
  - 心脏变为可攻击状态（`heartExposed = true`）
  - 心脏开始脉动（scale 缩放动画）
  - 显示提示文字："心脏暴露了！"
- 心脏被攻击时，每次碰撞-1HP，反弹球速度450
- 心脏HP=0时，触发 `clearLevel()`

## 四、器官状态系统

每个器官有三种可能的状态叠加：

### 4.1 状态定义

```
正常状态 ←→ 点燃状态（burnTimer > 0）
正常状态 ←→ 冻结状态（freezeTimer > 0）
点燃+冻结可同时存在
```

### 4.2 状态对器官的影响

| 状态 | 视觉 | 对器官运作的影响 | 对HP的影响 |
|------|------|------------------|------------|
| 正常 | 原色 | 正常运作 | 无 |
| 点燃 | 闪烁橙红 | 不影响运作 | 每秒-1HP，持续3秒 |
| 冻结 | 变冰蓝 | 完全停止运作 | 无 |
| 点燃+冻结 | 冰蓝+橙红闪烁 | 停止运作 | 每秒-1HP |

### 4.3 状态更新逻辑

在 `update()` 中新增 `updateOrganStates(delta)`：
1. 遍历所有存活器官
2. 如果 `burnTimer > 0`：递减，每满1000ms扣1HP，检查是否死亡
3. 如果 `freezeTimer > 0`：递减，到0时恢复运作
4. 点燃和冻结独立计时，互不干扰

## 五、弹珠选择与发射系统

### 5.1 弹珠库存

```
基础弹珠: 5颗
烈焰弹珠: 3颗
寒冰弹珠: 3颗
总计: 11颗（与现有第三关15颗相比减少，因为每种弹珠更强）
```

### 5.2 选择机制

- 球在发射台时（`isLaunching = true`），按 `1`/`2`/`3` 切换弹珠款式
- 切换时更新球的视觉（颜色、大小）和物理属性（弹力、半径）
- UI显示当前选中的弹珠款式和剩余数量
- 如果某款式数量为0，不可切换到该款式
- 发射后（`isLaunching = false`），弹珠款式锁定，直到球丢失或耐久归零

### 5.3 弹珠属性应用

切换弹珠款式时，实时修改：
- `ball.radius` + `ballBody.setCircle(radius)`
- `ballBody.setBounce(bounce)`
- `ballDurability` 重置为对应款式的耐久值
- `ballType` 变量记录当前款式（用于碰撞时决定伤害和特殊效果）

### 5.4 球丢失后

- 球掉落或耐久归零后，回到发射台
- `isLaunching = true`，弹珠款式重置为默认（基础弹珠，如果还有的话）
- 玩家重新选择款式并发射
- 所有款式都用完 → `lose()`

## 六、怪物整体反馈系统

### 6.1 怒吼机制

每当一个器官被摧毁时：
- 屏幕震动（`cameras.main.shake(300, 0.02)`）
- 所有存活器官的运作速度临时提升20%，持续3秒
- 显示浮动文字："怪物怒吼了！"

### 6.2 怪物HP总条

- 屏幕顶部显示一个总HP条，代表怪物整体剩余HP百分比
- 总HP = 所有器官HP之和（含心脏）
- 每次器官受损/被摧毁时更新
- 视觉上让玩家直观感受到怪物的"生命值"

### 6.3 心跳脉动

- 怪物存活时，整个地图背景以心跳节奏轻微脉动（scale 1.0↔1.01）
- 器官被摧毁越多，心跳越快（视觉紧张感递增）
- 心脏暴露后，心跳脉动幅度增大

在发射台附近显示三个弹珠款式图标：

### 9.2 怪物总HP条

## 十一、潜在风险与注意事项

### 11.1 已知技术限制（来自现有代码经验）

1. **Arcade Physics 不支持旋转体碰撞**：关节（旋转挡板）仍需手动线段碰撞检测，现有 `checkSpinnerCollision` 逻辑可复用
2. **场景重启需重置所有实例属性**：`resetState()` 必须清零所有新增字段（burnTimer、freezeTimer、organList等）
3. **背景标签页游戏循环不运行**：测试时需手动调用 `update()` 或在前台标签页测试
4. **`scene.restart()` 复用同一场景对象**：所有数组（organs、bullets等）必须在 `resetState()` 中清空
5. **Text对象纹理在场景关闭时销毁**：状态图标等动态创建的Text需在器官被摧毁时及时清理引用

### 11.2 新增风险点

1. **器官被摧毁后物理体残留**：必须同时 `destroy()` 游戏对象和从 `physics.add.collider` 中移除。现有代码用 `collider` 回调但没有保存collider引用——需改为保存collider引用，摧毁时 `collider.destroy()`
2. **灼烧伤害在器官已死亡后仍触发**：`updateOrganStates()` 中每次扣HP后需检查 `if (organ.hp <= 0) destroyOrgan(organ)`，并停止后续灼烧
3. **弹珠款式切换时物理属性未完全重置**：切换时需同时更新 `radius`、`bounce`、`body.setCircle()`、`body.setBounce()`、视觉颜色和大小
4. **心脏暴露条件判断时机**：应在每次器官被摧毁后立即检查，而非每帧检查（避免重复触发暴露动画）
5. **冲击波影响范围判断**：脊椎冲击波需遍历所有球（目前只有1个球，但逻辑上应支持多球），用距离判断是否在范围内
6. **冻结状态与碰撞冷却的交互**：冻结的器官不造成球耐久伤害，但物理碰撞（弹射）仍应生效——需在碰撞回调中检查 `freezeTimer` 来决定是否调用 `damageBall()`

### 11.3 性能考量

- 13个器官 + 心脏 + 子弹 + 状态图标 ≈ 30个活跃游戏对象，性能无压力
- 灼烧/冻结状态更新在 `update()` 中遍历器官列表，13个对象的遍历开销可忽略
- 冲击波每5秒触发一次，影响范围判断仅对球（1个对象），无性能问题

## 十二、总结

| 维度 | 现有设计 | 新设计 |
|------|----------|--------|
| Boss | 上方1~2个圆形Boss | 地图即怪物，13个器官+心脏 |
| 弹珠 | 1种基础弹珠 | 3种：基础/烈焰/寒冰 |
| 伤害 | 统一1点 | 基础1/烈焰2(+DoT)/寒冰1(+冻结) |
| 目标 | 击败Boss HP | 摧毁75%器官→暴露心脏→摧毁心脏 |
| 路径 | 固定地图 | 器官被摧毁后路径动态变化 |
| 策略 | 纯物理操作 | 弹珠款式选择+器官优先级决策 |
| 反馈 | Boss闪烁 | 怒吼+心跳+总HP条+状态图标 |

**核心体验转变**：从"用挡板把球弹向Boss"变为"策略性地选择弹珠款式，按优先级摧毁怪物器官，管理有限弹珠资源，在路径不断崩塌的怪物体内找到通往心脏的路"。
